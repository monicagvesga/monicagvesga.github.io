Routing tickets in Zendesk, step by step.

How to get each case to the right agent fast, so you cut reassignments (the bounce), speed up resolution, and lift CSAT. Four setups, with the exact path to each one.

Free to use · by Mónica Gómez Vesga
About these imagesThese are simplified mockups of each screen, drawn to keep the steps clear. They are not real Zendesk screenshots. Each step links to Zendesk's official guide, where you'll see the live screens.
1

Sort the ticket the moment it lands

A trigger runs on every new ticket. Use one to send it to the right group, set its priority, and tag it, so it starts in the correct queue instead of bouncing around.

Admin Center › Objects and rules › Business rules › Triggers › Create trigger
  1. 1
    Conditions decide which tickets this trigger catches, by channel, language, topic, tags, even sentiment.
  2. 2
    Actions send it to the right group and set its priority, so it lands in the correct queue from the start.
  3. 3
    The auto_routing tag is what hands the ticket over to omnichannel routing (Step 3) for auto-assignment.

Zendesk's official guide: Routing incoming support requests ↗

2

Route by skill, not just who's free

Skills are agent attributes (language, product, billing, technical). Tag a ticket with the skill it needs and it goes to an agent who can actually solve it, so the first person to open it is the right person.

Admin Center › Objects and rules › Business rules › Skills
  1. 1
    Create skill types and skills (Language, Product, Technical, Billing).
  2. 2
    Assign each skill to the agents who have it.
  3. 3
    Use a trigger to add the needed skill to the ticket. Mark it required or optional. Required never times out; optional is dropped if no skilled agent is free, so the ticket still moves.

Zendesk's official guide: Using skills to route tickets ↗

3

Let omnichannel routing balance the load

Turn this on and Zendesk auto-assigns tickets across email, chat, and calls by who is available and has spare capacity, so nothing sits unassigned and no single agent drowns. On higher plans it also weighs priority and skills.

Admin Center › Objects and rules › Omnichannel routing
  1. 1
    Turn it on. It works with the triggers you set in Step 1; email tickets need the auto_routing tag to be picked up.
  2. 2
    Pick the assignment method: spare capacity (busiest gets less) or round robin (take turns).
  3. 3
    Set capacity rules so each agent only holds so much at once. That is what stops anyone from drowning.

Zendesk's official guide: About omnichannel routing ↗

4

Prioritize with SLAs and clean views

An SLA policy sets time targets per priority, and a good view surfaces the most urgent and oldest tickets first, so nothing slips while the team stays fast.

Admin Center › Objects and rules › Business rules › Service level agreements
  1. 1
    Set time targets per priority (first reply, next reply, resolution). Urgent gets a tighter clock than Normal.
  2. 2
    Build a view sorted by priority and age, so the most urgent and oldest tickets sit at the top and nothing gets buried.

The payoff. Route it right the first time and you cut reassignments and reopens (the bounce), speed up first response and resolution, and that lands directly in your CSAT.

A note on plans: triggers, groups, and views work on every Zendesk plan; skills- and priority-based routing live on the higher tiers.
